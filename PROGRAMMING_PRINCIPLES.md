# Programming Principles Implementation

У цьому проєкті реалізовано ієрархію класів для музичних інструментів. При розробці було дотримано ключових принципів програмування та об'єктно-орієнтованого дизайну.


## 1. Inheritance (Успадкування)
Це принцип, що дозволяє створювати нові класи на основі існуючих. У проєкті базовий абстрактний клас `MusicalInstrument` визначає загальну структуру, яку успадковують конкретні інструменти.

* [Базовий клас MusicalInstrument](./MusicalInstrument.cs#L5)
* [Успадкування класу Violin](./Instruments.cs#L5)

## 2. Polymorphism (Поліморфізм)
Принцип, який дозволяє використовувати об'єкти похідних класів через посилання на базовий клас. У методі `Main` створено масив `MusicalInstrument[]`, який містить різні інструменти, але дозволяє викликати їх методи в єдиному циклі.

* [Масив об'єктів різних типів](./Program.cs#L16-L22)
* [Однотипний виклик методів у циклі](./Program.cs#L25-L33)

## 3. Abstraction (Абстракція)
Використання абстрактних методів дозволяє визначити "контракт" для всіх нащадків. Ми знаємо, що кожен інструмент має звук (`Sound`), але як саме він звучить — вирішує конкретний клас.

* [Оголошення абстрактних методів](./MusicalInstrument.cs#L38-L41)
* [Реалізація методу Sound у класі Trombone](./Instruments.cs#L15)

## 4. Encapsulation (Інкапсуляція)
Приховання внутрішніх даних класу. Поля `Name` та `Characteristics` мають модифікатор `protected`, що обмежує прямий доступ до них ззовні, але дозволяє використовувати їх у класах-нащадках. Доступ до імені реалізовано через методи (геттери/сеттери).

* [Захищені поля класу](./MusicalInstrument.cs#L7-L8)
* [Методи SetName та GetName](./MusicalInstrument.cs#L44-L45)
